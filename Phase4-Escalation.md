# Phase 4: Escalation (Authentication & Lateral Movement Attempts)

This phase documents observed post-compromise activity where the attacker transitions from web-layer interaction toward system-level access attempts. The focus is on authentication anomalies and early-stage lateral movement indicators.

---

# Phase Objective

To identify and analyze attempts aimed at expanding control beyond the web application context into underlying system services, particularly SSH authentication pathways.

---

# Observed Activity

Following web-based interaction, the source IP began generating authentication requests targeting the SSH service on the host system.

These attempts were characterized by:
- invalid usernames
- repeated login failures
- rapid authentication bursts

# Correlation Insight

This activity correlates with earlier web-layer interaction from the same source IP, indicating progression from:
- application interaction → system service probing

This suggests early-stage lateral movement behavior.

---

# Detection Indicators

The following behavioral patterns were identified:

- repeated SSH authentication failures
- invalid username enumeration
- consistent source IP reuse
- rapid login attempt sequencing

# Hash-Check

The SHA256 hash generated by the Wazuh File Integrity Monitoring system for the uploaded file was later analyzed using VirusTotal for external reputation validation. The hash returned multiple malicious indicators across antivirus engines, confirming that the file exhibited known web shell–like behavior. This external intelligence correlation strengthened the internal SIEM alert, validating the detection as a high-confidence compromise indicator rather than a false positive.

# Summary

Phase 4 highlights escalation attempts from web-based interaction into system authentication probing. The correlation of SSH failures with prior web activity strengthens the conclusion of a single coordinated intrusion sequence.
